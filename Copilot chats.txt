import re

def parse_ios_config(text: str) -> dict:
    data = {"mgmt": {}, "l2": {"vlans": set()}, "l3": {}, "security": {}}

    # SSH/Telnet
    data["mgmt"]["ssh"] = bool(re.search(r"\bip ssh version\b", text))
    data["mgmt"]["telnet_enabled"] = bool(re.search(r"transport input telnet", text))

    # VLANs
    vlans = re.findall(r"^vlan\s+(\d+)", text, flags=re.M)
    data["l2"]["vlans"] = set(vlans)

    # Password encryption
    data["security"]["service_password_encryption"] = bool(
        re.search(r"service password-encryption", text)
    )
    return data

rules_engine.py ‚Äî evaluate rules

import yaml

def evaluate(config_dict, rules_path="rules"):
    results = []
    for rule_file in ["ssh_telnet.yaml", "vlan_standard.yaml", "password_encryption.yaml"]:
        rule = yaml.safe_load(open(f"{rules_path}/{rule_file}"))
        ok = True

        # simple checks
        if rule["id"] == "SEC-SSH-001":
            ok = config_dict["mgmt"]["ssh"] and not config_dict["mgmt"]["telnet_enabled"]

        if rule["id"] == "SEC-PASS-ENC-001":
            ok = config_dict["security"]["service_password_encryption"]

        results.append({"id": rule["id"], "title": rule["title"], "pass": ok, "severity": rule["severity"]})
    return results

Telnet should be turned off because it is inherently insecure compared to modern alternatives like SSH. Here‚Äôs why:

‚úÖ 1. Telnet Sends Data in Plain Text
All communication‚Äîincluding usernames and passwords‚Äîis transmitted unencrypted.
This means anyone who intercepts the traffic (via packet sniffing or man-in-the-middle attack) can read sensitive information easily.
Orchard Program -CIS
‚úÖ 2. Vulnerable to Eavesdropping & Attacks
Because Telnet lacks encryption, attackers can capture credentials and session data.
It opens the door to session hijacking and unauthorized access.
network_clients_linux
‚úÖ 3. Weak Authentication
Telnet uses basic authentication methods, often without encryption.
SSH, by contrast, supports strong password and key-based authentication.
Orchard Program -CIS
‚úÖ 4. Increased Attack Surface
Leaving Telnet enabled is considered an unused legacy service.
Best practice: disable Telnet and other obsolete protocols (like FTP) to reduce attack vectors.
network_clients_linux
‚úÖ Real-Time Example
Imagine you have a router in a branch office using Telnet for remote access.
An attacker on the same network runs a packet sniffer and captures:

Username: admin
Password: admin123
Because Telnet sends this in plain text, the attacker now has full control of your device.
If SSH were used, the credentials and session would be encrypted, making interception useless.

Bottom line: Telnet is outdated and insecure. Always use SSH for remote administration because it encrypts data, supports strong authentication, and protects against eavesdropping.
